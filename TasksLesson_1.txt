Library part 1
Задача 1

Попробуем автоматизировать библиотеку.
На полках в библиотеке находится много печатных изданий. У них есть общие характеристики - название (name) и страницы.

1. Создадим класс родитель для всех изднаий в библиотеке - Publication, в котором у нас будут общие свойстав:
name - название публикации
countPages = количество страниц в книге

2. Необходимо реализовать конструктор, который принимает на вход два параметра - name и countPages

3. Необходимо реализовать метод для печати публикаций, print(), который будет возвращать строку в формате
"name=<название_публикации>,countPages=<количество_страниц>";

Задача 2

Добавляется класс Book, он является дочерним классом для Publication. Для книг характерно поле author, также у книг есть поля name - название книги и countPages - количество страниц.

1. Создать класс Book

2. Создать конструктор, который будет принимать name - название книги, countPages - количество страниц, author - имя/фамилия автора книги.

3. Реализовать метода print, который будет возвращать строку в формате "Book{name=<название_книги>,countPages=<количество_страниц>,author=<автор_книги>}"


Задача 3.

Добавим журналы, класс Journal. Он расширяет класс Publication и добавляет новые свойства - год журнала и номер журнала.

1. Создать класс Journal, который расширяет Publication

2. Создать конструктор, который будет принимать 4 параметра:
    name - название журнала
    countPages - количество страниц
    number - номер журнала
    publicationYear - год публикации

3. Реализовать метод print, который будет возвращать строку - "Journal{name=<название_журнала>,countPages=<количество_страниц>,number=<номер_журнала>,year=<год_публикации>}"

Задача 4.

Для автоматизации библиотеки, необходимо создать класс Library, который печатает информацию про каждую публикацию, которое находится в переданном массиве публикаций.

На выходе ожидается вывод всех изданий в консоль.
Если количество элементов в массиве = 0, напечатать "Нет публикаций для печати".

1. Реализовать класс Library

2. Реализовать метод printPublciations(), который принимает массив Publication[]. Данный метод должен печатать публикации в консоль.

Задача 5.

Попробуем создать хранилище для нашей библиотеки, что бы была возможность наполнять
библиотеку новыми Журналами и книгами.
Под хранилищем будем воспринимать класс, в котором будет храниться массив всех публикаций.
У хранилища должны быть 2 конструктора, конструктор по умолчанию будет создавать хранилище с
размером по умолчанию в 16 элементов.
Конструктор с параметром int size, будет создавать хранилище с размером size.


Задача 6.
